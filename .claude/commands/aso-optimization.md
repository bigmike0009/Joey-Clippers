# ASO Optimization

Optimize App Store presence for: $ARGUMENTS

Read `docs/PRD.md` for app description and target user context if available.

## Step 1: Keyword Research

Identify high-value keywords for this app:

1. **Core keywords**: terms that directly describe what the app does
2. **Problem keywords**: terms users search when they have the problem this app solves
3. **Competitor keywords**: terms users search for similar apps
4. **Long-tail keywords**: specific phrases with lower competition

For each keyword, evaluate:
- Search volume (high/medium/low estimate)
- Competition level (how many quality apps rank for this term?)
- Relevance to the app's core functionality

## Step 2: App Name & Subtitle

**App Name** (30 chars max):
- Lead with the brand name
- Append the primary keyword if it fits naturally
- Example: "Habitica: Habit & Task Tracker"

**Subtitle** (30 chars max):
- Use different keywords than the name
- Describe the primary benefit, not just the feature
- Example: "Build habits with your friends"

## Step 3: Keyword Field (100 chars, iOS only)

- Comma-separated, no spaces after commas
- No keywords already in the name or subtitle (redundant)
- No competitor app names (against guidelines)
- Include variations, synonyms, and related terms

## Step 4: Description (First 3 Lines Are Critical)

The first ~170 characters appear before "More":
- Lead with the primary benefit statement
- Address the target user's problem directly
- Include 1–2 high-value keywords naturally

Body structure:
```
[Hook — problem/benefit statement]

KEY FEATURES:
• [Feature 1 — benefit-first]
• [Feature 2 — benefit-first]
• [Feature 3 — benefit-first]

[Social proof or secondary pitch if applicable]

[Call to action]
```

## Step 5: Screenshots

- Screenshot 1: The core value proposition (most important moment in the app)
- Screenshot 2–3: The key features that differentiate from competitors
- Screenshots 4–6: Supporting features, social proof, or secondary flows
- Each screenshot should have a caption (2–5 words, benefit-focused)

## Output

```
**App Name:** [30 chars]
**Subtitle:** [30 chars]
**Keywords:** [100 chars]
**Description (first 170 chars):** [text]
**Screenshot captions:** [1] [2] [3] [4] [5]
```

Explain the keyword strategy and why each choice was made.
